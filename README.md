# 🎬 MovieNest

Full-stack каталог дорам: React (Vite) на фронтенде, Express на бэкенде. Каталог фильмов — обычный JSON-файл (`backend/data/movies.json`), а данные пользователей (аккаунты, XP, лидерборд, достижения, комментарии, прогресс просмотра) хранятся в MongoDB — так они переживают перезапуск/пересборку/"засыпание" сервера на бесплатном хостинге (см. DEPLOY.md).

> **Важно:** сайт хранит ровно 10 дорам в `backend/data/movies.json`. Добавлять, редактировать и удалять записи можно либо вручную (см. ниже), либо через раздел «Контент» в админ-панели (`/admin`, доступен только администраторам).

## Быстрый старт

Из корневой папки `MovieNest`:

```bash
npm install
npm run dev
```

Этого достаточно. Команда `npm install` установит зависимости и фронтенда, и бэкенда (через npm workspaces), а `npm run dev` одновременно запустит:

- **Backend** на `http://localhost:5000`
- **Frontend** на `http://localhost:5173`

Откройте в браузере **http://localhost:5173**.

> Frontend настроен с proxy (`vite.config.js`), поэтому все запросы `/api/...` автоматически уходят на backend — отдельно настраивать ничего не нужно.

### Запуск по отдельности (если нужно)

```bash
npm run dev:backend    # только сервер
npm run dev:frontend   # только клиент
```

## Как добавить свою дораму (без программирования)

Проще всего — через админ-панель: войдите под одним из 4 email-адресов из `ADMIN_EMAILS` (см. раздел про роли ниже), откройте `/admin` → вкладка «Контент (дорамы)» → «+ Добавить дораму». Форма сама проверит обязательные поля и сохранит запись.

Вручную это тоже можно сделать, открыв файл:

```
backend/data/movies.json
```

Это обычный текстовый JSON-файл. Скопируйте один из существующих блоков `{ ... }` и измените значения:

```json
{
  "id": 11,
  "slug": "moya-dorama",
  "title": "Название дорамы",
  "description": "Краткое описание сюжета.",
  "poster": "https://ссылка-на-картинку-постера.jpg",
  "rating": 7.5,
  "year": 2024,
  "genre": ["Дорама", "Драма"],
  "type": "dorama",
  "country": "Корея",
  "episodesCount": 16,
  "episodes": [{ "number": 1, "title": "Серия 1", "videoUrl": "" }],
  "recommended": false,
  "popular": false,
  "newRelease": true,
  "trailerUrl": "ВСТАВЬТЕ ССЫЛКУ НА ВИДЕО ЗДЕСЬ"
}
```

Пояснение полей:

| Поле            | Значение                                                          |
|-----------------|--------------------------------------------------------------------|
| `id`            | Уникальное число (не повторяется у других записей)                |
| `slug`          | Латиницей, без пробелов — используется в адресе `/movie/slug`     |
| `title`         | Название                                                            |
| `description`   | Описание, которое видно на странице дорамы                        |
| `poster`        | Прямая ссылка на изображение постера                               |
| `rating`        | Рейтинг, число от 0 до 10                                          |
| `year`          | Год выхода                                                          |
| `genre`         | Список жанров — используется в фильтрах                            |
| `type`          | Всегда `"dorama"`                                                  |
| `episodesCount` | Число серий — должно совпадать с длиной массива `episodes`         |
| `episodes`      | Список серий: `number`, `title`, `videoUrl`                        |
| `recommended`   | `true`, если показывать в блоке «Рекомендуемые» на главной         |
| `popular`       | `true`, если показывать в блоке «Популярные»                       |
| `newRelease`    | `true`, если показывать в блоке «Новинки»                          |
| `trailerUrl`    | Ссылка на видео/трейлер. Пустая строка `""`, если видео ещё нет    |

После сохранения файла обновите страницу в браузере — новая дорама появится автоматически. **Перезапускать сервер не нужно.**

Если `trailerUrl` и `videoUrl` эпизода пустые — при нажатии «Смотреть» пользователь увидит сообщение «Видео пока недоступно».

## Технологии

- **Frontend:** React 18, Vite, React Router, чистый CSS (без UI-библиотек)
- **Backend:** Node.js, Express.js
- **База данных:** MongoDB (пользователи, XP, лидерборд, достижения, челленджи, комментарии, прогресс просмотра, ежедневные входы) + `backend/data/movies.json` для каталога фильмов (статический контент, редактируется через админ-панель)

## ⚠️ Обязательная настройка перед первым запуском

Проект использует авторизацию по паролю, поэтому нужен секретный ключ для подписи токенов.

```bash
cp backend/.env.example backend/.env
```

Затем откройте `backend/.env` и замените `JWT_SECRET` на длинную случайную строку. Сгенерировать её можно командой:

```bash
node -e "console.log(require('crypto').randomBytes(48).toString('hex'))"
```

Без этого шага сервер откажется запускаться (специально, чтобы никто случайно не выложил сайт в интернет с предсказуемым секретом).

Для фронтенда `.env` не обязателен в разработке (Vite сам проксирует `/api` на backend), но пример есть в `frontend/.env.example` на случай продакшен-деплоя на разные домены.

## 👑 Роли: admin и user

В системе всего 2 роли:

- **admin** — полный доступ к админ-панели (`/admin`): статистика всех пользователей + добавление, редактирование и удаление дорам.
- **user** — обычный зарегистрированный пользователь: избранное, достижения, задания, лидерборд, комментарии.

Кто становится админом, задаётся переменной `ADMIN_EMAILS` в `backend/.env` — перечислите через запятую **свой email и email'ы трёх друзей** (максимум 4 адреса, остальные будут проигнорированы):

```
ADMIN_EMAILS=you@example.com,friend1@example.com,friend2@example.com,friend3@example.com
```

Как только человек с одним из этих email регистрируется (или уже зарегистрирован и входит в аккаунт), он автоматически получает роль `admin`. Все остальные — только `user`. Роль нельзя получить или изменить никаким другим способом (ни через тело запроса регистрации, ни через профиль) — это исключает повышение привилегий обычным пользователем.

## Новые возможности (аккаунты, достижения, лидерборд)

- **Регистрация и вход** — Email + пароль, пароль хранится только в виде bcrypt-хэша. Сессия хранится в HttpOnly cookie на 30 дней — пользователь остаётся в аккаунте после F5 и перезапуска браузера, пока не нажмёт «Выйти».
- **Профиль** (`/profile`) — аватар, Email, дата регистрации, число просмотров, уровень, достижения.
- **Достижения** — Bronze (10), Silver (20), Gold (40), Legend (50) просмотров. При получении показывается анимированная модалка + уведомление.
- **Таблица лидеров** (`/leaderboard`) — топ пользователей по числу просмотров, ТОП-3 выделены отдельно.
- **Засчитывание просмотров** — засчитывается только после ~25 секунд реального нахождения на странице просмотра (видна вкладка), и только один раз за фильм на пользователя — обновление страницы или повторное открытие того же фильма счётчик не увеличивают.
- **Комментарии** к фильмам — на странице просмотра, только для авторизованных пользователей.
- **Безопасность** — bcrypt, JWT в HttpOnly cookie, CSRF-защита (double-submit cookie), Helmet с CSP, rate limiting (в т.ч. защита от подбора пароля с временной блокировкой аккаунта), валидация всех входных данных через express-validator.

Все данные (пользователи, просмотры, достижения, комментарии) хранятся в `backend/data/*.json`, как и каталог фильмов — отдельная СУБД не требуется.
